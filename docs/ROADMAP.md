## 개발 원칙

- 각 단계는 **이전 단계가 완료된 후** 시작

## Phase 1 · 프로젝트 초기 설정 (골격 구축)

> **이유**: 타입과 라우트 구조가 없으면 이후 작업이 서로 충돌한다. 먼저 뼈대를 세워야 팀원(또는 다음 세션)이 병렬로 작업할 수 있다.

**예상 소요 시간**: 1~2시간

### 작업 목록

| # | 작업 | 파일 |
|---|------|------|
| 1.1 | 데이터 타입 정의 | `types/invoice.ts` |
| 1.2 | JSON 스토어 초기화 | `lib/store.ts`, `data/invoices.json` |
| 1.3 | 라우트 골격 생성 (빈 페이지) | `app/invoices/page.tsx`<br>`app/invoices/new/page.tsx`<br>`app/invoices/[id]/page.tsx`<br>`app/invoices/[id]/edit/page.tsx`<br>`app/invoices/[id]/print/page.tsx` |
| 1.4 | 필요한 ShadcnUI 컴포넌트 설치 | `table`, `form`, `input`, `label`, `select`, `textarea`, `separator`, `tooltip`, `dialog` |
| 1.5 | 루트 리다이렉트 설정 | `app/page.tsx` → `/invoices` |

### 완료 기준

- [ ] `types/invoice.ts` — `Invoice`, `InvoiceItem`, `InvoiceStatus` 타입 정의 완료
- [ ] `data/invoices.json` — 빈 배열 `[]` 파일 존재
- [ ] 모든 라우트 경로에 최소한의 placeholder 페이지 존재 (404 없음)
- [ ] `npm run build` 에러 없음

> **이유**: Server Actions와 유틸 함수는 모든 페이지가 공유한다. 핵심 기능 개발 전에 완성해야 중복 코드와 불일치를 막을 수 있다.

**예상 소요 시간**: 1~2시간

### 작업 목록

| # | 작업 | 파일 |
|---|------|------|
| 2.1 | Server Actions 전체 구현 | `app/actions/invoice.ts`<br>(`createInvoice`, `updateInvoice`, `deleteInvoice`, `getInvoice`, `listInvoices`) |
| 2.2 | 견적서 번호 생성 유틸 | `lib/utils.ts` 또는 `lib/invoice-utils.ts`<br>(`INV-YYYYMMDD-{id 앞 4자리}` 형식) |
| 2.3 | 금액 계산 유틸 | 소계 · 부가세 · 합계 계산 함수 |

## Phase 3 · 핵심 기능 개발

> **이유**: MVP의 핵심 유저 스토리(작성 → 열람)를 먼저 완성해야 한다. 목록보다 작성 폼을 먼저 만들어야 테스트 데이터를 생성할 수 있고, 뷰 페이지는 공유 URL의 핵심이다.

**예상 소요 시간**: 4~6시간

### 3-A. 견적서 작성 폼 (`/invoices/new`)

| # | 작업 |
|---|------|
| 3A.1 | 발행자 정보 필드 (회사명, 연락처, 이메일) |
| 3A.2 | 수신자 정보 필드 (클라이언트명, 연락처) |
| 3A.3 | 견적 제목 · 유효기간 · 비고 필드 |
| 3A.4 | 품목 리스트 동적 추가/삭제 (품목명, 수량, 단가 → 공급가액 자동 계산) |
| 3A.5 | 소계/부가세/합계 실시간 계산 패널 |
| 3A.6 | 저장 버튼 → `createInvoice` Server Action → 목록 이동 |

### 3-B. 견적서 웹 뷰 (`/invoices/[id]`)

| # | 작업 |
|---|------|
| 3B.1 | A4 비율 카드 레이아웃 |
| 3B.2 | 상단 헤더 (회사명, 견적서 번호, 발행일) |
| 3B.3 | 발행자/수신자 정보 블록 |
| 3B.4 | 품목 테이블 (품목명 · 수량 · 단가 · 공급가액) |
| 3B.5 | 소계/부가세/합계 섹션 |
| 3B.6 | 비고 섹션 |
| 3B.7 | 우측 상단 고정 액션 바 (PDF 다운로드, 링크 복사 버튼 플레이스홀더) |

### 3-C. 견적서 목록 (`/invoices`)

| # | 작업 |
|---|------|
| 3C.1 | 카드 또는 테이블로 전체 목록 표시 |
| 3C.2 | 각 행: 제목, 클라이언트명, 합계, 상태 배지, 작성일 |
| 3C.3 | "새 견적서 작성" 버튼 → `/invoices/new` |
| 3C.4 | 행 클릭 → `/invoices/[id]` |

### 완료 기준

- [ ] 견적서 작성 → 저장 → 목록에 표시되는 전체 흐름 동작
- [ ] 공유 URL(`/invoices/[id]`) 직접 접속 시 견적서 정상 렌더링

## Phase 4 · 추가 기능 개발

> **이유**: 핵심 열람 흐름이 완성된 후에 부가 기능을 붙인다. PDF는 뷰 레이아웃이 확정되어야 정확히 구현할 수 있고, 수정 폼은 작성 폼 컴포넌트를 재사용한다.

**예상 소요 시간**: 3~4시간

### 4-A. PDF 다운로드

| # | 작업 |
|---|------|
| 4A.1 | `/invoices/[id]/print` 인쇄 전용 레이아웃 (헤더/푸터/네비 숨김) |
| 4A.2 | `@media print` CSS — A4 용지 최적화 |
| 4A.3 | 액션 바 "PDF 다운로드" 버튼 → `window.open('/invoices/[id]/print')` 후 `window.print()` |
| 4A.4 | 파일명 `견적서_{clientName}_{id}.pdf` 설정 |

### 4-B. 견적서 수정 폼 (`/invoices/[id]/edit`)

| # | 작업 |
|---|------|
| 4B.1 | 기존 데이터 로드 후 작성 폼 컴포넌트 재사용 |
| 4B.2 | 저장 → `updateInvoice` Server Action |
| 4B.3 | 삭제 버튼 → `deleteInvoice` → 목록 이동 |

### 4-C. 공유 링크 복사

| # | 작업 |
|---|------|
| 4C.1 | 목록 페이지 공유 링크 복사 버튼 (클립보드 API) |
| 4C.2 | 웹 뷰 액션 바 "링크 복사" 버튼 |
| 4C.3 | 복사 성공 시 토스트 또는 툴팁 피드백 |

### 완료 기준

- [ ] "PDF 다운로드" 클릭 시 브라우저 인쇄 다이얼로그 표시
- [ ] 인쇄 레이아웃에서 헤더/푸터/버튼 숨겨짐
- [ ] 수정 폼에서 기존 데이터 불러와 저장 후 반영
- [ ] 링크 복사 버튼 클릭 시 클립보드에 URL 저장
